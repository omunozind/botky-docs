Este tipo de acción le permite a tu bot enviar notificaciones por distintos canales: correo, Slack, SMS, llamada telefónica, WhatsApp, Telegram, Viber, Facebook, o directamente al chat en vivo.

<Tabs>
  <Tab title="Correo">
    Requiere tener una integración de correo configurada primero.

    <Card type="note">
      A diferencia del paso "Enviar correo", la notificación por correo **solo admite mensajes de texto**.
    </Card>

    Configura el remitente en el campo **Perfil de correo**, y el destinatario en **Correo destino**.

    <Frame caption="Configuración de una notificación por correo">
      <Placeholder />
    </Frame>
  </Tab>

  <Tab title="Slack">
    Requiere una integración de Slack ya configurada.

    <Steps>
      <Step title="Elige un Webhook">
        En la ventana emergente.
      </Step>
      <Step title="Elige el canal">
        Puedes usar el canal por defecto o indicar uno distinto.
      </Step>
      <Step title="Elige un tipo de mensaje">
        Puedes agregar tantos como necesites — por ejemplo, el tipo "Section".
      </Step>
      <Step title='Completa el contenido (tipo "Section")'>
        Escribe tu mensaje en el área de contenido, y agrega foto o valores de campo si lo necesitas.
      </Step>
      <Step title="Prueba y guarda">
        Usa el botón "Test" para enviar un mensaje de prueba y ajustar lo que haga falta. Cuando esté listo, haz clic en "Guardar".
      </Step>
    </Steps>

    <Frame caption="Configuración de una notificación de Slack">
      <Placeholder />
    </Frame>
  </Tab>

  <Tab title="SMS">
    Requiere un número remitente ya configurado. Elige remitente y destinatario, escribe el mensaje, y listo.

    <Card type="tip">
      Puedes combinar esto con la operación "generar texto aleatorio" sobre una variable de texto, para enviar un código de verificación o seguridad generado al momento.
    </Card>

    <Frame caption="Configuración de una notificación SMS">
      <Placeholder />
    </Frame>
  </Tab>

  <Tab title="Llamada telefónica">
    Requiere un número de teléfono y un **flujo de voz conectado a ese número**, para que el bot de voz tenga con qué responder.

    Usa el campo **Ref Payload** para pasarle datos al flujo de voz — ese valor debe guardarse en un campo personalizado en el Paso Inicial del Flujo Principal, para que el flujo de voz pueda leerlo.

    <Card type="tip">
      Puedes pasar una variable JSON completa en el Ref Payload, pero necesitas codificarla y decodificarla en base64 antes y después — revisa las operaciones de base64 disponibles al fijar un campo personalizado.
    </Card>

    <Frame caption="Configuración de una notificación de llamada telefónica">
      <Placeholder />
    </Frame>
  </Tab>

  <Tab title="WhatsApp">
    Requiere tener la integración de WhatsApp configurada. Elige la cuenta de WhatsApp y envía tu mensaje — recuerda que el número del destinatario debe incluir el código de país.

    <Card type="warning">
      El destinatario debe haber interactuado contigo en las últimas **24 horas** — si no, el mensaje podría fallar al entregarse.
    </Card>

    <Frame caption="Configuración de una notificación de WhatsApp">
      <Placeholder />
    </Frame>
  </Tab>

  <Tab title="Plantilla de WhatsApp">
    Requiere la integración de WhatsApp **y un método de pago verificado** asociado a tu número. Selecciona la plantilla y completa sus parámetros.

    Si la plantilla tiene un botón de respuesta rápida, hay dos comportamientos posibles al hacer clic:

    <CardGroup cols={2}>
      <Card title="El remitente continúa" icon="arrow-right">
        El usuario original (A) avanza al siguiente paso una vez que el administrador (B) recibe la notificación y hace clic en el botón.
      </Card>
      <Card title="El receptor continúa" icon="arrow-right-to-bracket">
        El administrador (B) avanza a su siguiente paso al hacer clic en el botón — no el usuario original.
      </Card>
    </CardGroup>

    <Card type="note">
      Ejemplo: el usuario A conversa con tu bot y llega al paso de notificación con plantilla, que tiene un botón, dirigida al administrador B. Si es "el remitente continúa", cuando B recibe el mensaje y hace clic, **A** sigue al siguiente paso. Si es "el receptor continúa", cuando B hace clic, es **B** quien sigue a su siguiente paso.
    </Card>

    <Frame caption="Configuración de una notificación con Plantilla de WhatsApp">
      <Placeholder />
    </Frame>
  </Tab>

  <Tab title="Telegram y Viber">
    Requiere la integración correspondiente (Telegram o Viber). Elige el bot, y filtra destinatarios por **User ID, correo, teléfono o User NS**.

    <Card type="note">
      A diferencia de WhatsApp, aquí **no existe la restricción de 24 horas**. Viber funciona igual que Telegram.
    </Card>

    <Frame caption="Configuración de una notificación de Telegram/Viber">
      <Placeholder />
    </Frame>
  </Tab>

  <Tab title="Facebook">
    Requiere la integración de Facebook. Usa etiquetas de mensaje para cumplir con las políticas de Meta: por defecto no se usa ninguna etiqueta, en cuyo caso el destinatario debe estar dentro de la ventana de 24 horas o el mensaje fallará. Elegir una etiqueta adecuada puede evitar esa restricción.

    <Frame caption="Configuración de una notificación de Facebook">
      <Placeholder />
    </Frame>
  </Tab>

  <Tab title="Chat en vivo">
    Envía una notificación de texto libre que aparece directamente dentro del Chat en vivo para tus agentes humanos. Requiere tener configurada una integración de chat en vivo que reciba el mensaje — solo el proveedor específicamente configurado la recibirá.

    <Frame caption="Notificación apareciendo en el Chat en vivo">

    </Frame>
  </Tab>
